---
title: "Deactivating Applications"
sidebar_position: 6
---

In Koodisi, you create and configure applications in the Development environment, and deploy them in the Production environment. Ideally, the Applications panel in the Development environment should list only those applications under active configuration.

Once you've deployed your application, you no longer need to update it. Koodisi enables you to deactivate such applications in the Development environment, removing them from the Available section in the Applications panel to the Inactive section, so users don't update them by mistake.

:::info
Deactivating applications in the Development environment doesn't deactivate them in the Production environment.
:::

To deactivate an application:

1. Click the **Manage Apps** icon in the upper-right corner of the Applications panel. The panel refreshes, displaying all workspace applications organized by Active and Inactive status. A selected check box appears against each application that's in Active status.
2. To make an application inactive, deselect the checkbox next to it. To make an inactive application available, select its checkbox.
3. Click **Save Changes**.

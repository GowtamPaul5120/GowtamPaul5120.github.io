New and Improved Features

1. **Data Table Enhancements**\
   Improved aggregation workflows with a conflicting modal to resolve data collisions and a toggle to summarize results. The behavior of the "distinct" operation has also been corrected when alias is omitted.
2. **Engage Enhancements**

* The _SendToEngage_ activity now includes default schema and target schema updates.
* Event names are now listed in the Engage configuration for better traceability.

3. **CSV Payload Support in Test Panel**\
   Added support for CSV format in test payloads, allowing better flexibility during debugging and validation.
4. **Tab and UI Improvements**

* Enhanced tab experience and layout interactions.
* Added common text truncation for all sidebar panel items.
* Resolved issues with panel resizing in the configuration view.

5. **Manifest and Connection Filtering**\
   Restored missing Manifest feature flags and applied filters to connection resources including SFTP, ABS, OAUTH, and S3.
6. **Git Integration**\
   Added support for switching branches within multi-module Git workspaces, enabling smoother context handling across modules.

#### Bug Fixes

1. **Global Variable Import**\
   Global variables now appear correctly during workflow import operations.
2. **Clear Mapped Context**\
   Introduced an option to clear mapped context in workflows.
3. **Schema Registry Rename Validation**\
   Fixed an issue where renaming a collection to an existing name was not properly handled.
4. **Commit Dropdown Issue**\
   Addressed a UI bug where the commit dropdown remained stuck and unresponsive after interaction.

#### Considerations

1. **Multitabs (Beta)**\
   Functionality updates are underway. Currently, a new tab can only be opened if the current selection has unsaved changes. In an upcoming release, users will be able to right-click any artifact and open it in a new tab directly—improving multitasking and navigation workflows.
2. **Git Application Setup**\
   If your Git-connected application appears empty or missing, please ensure a workflow is selected to initialize the Git context properly.

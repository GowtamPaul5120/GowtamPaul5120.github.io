## Key Features

### Workspace Governance Redesign

This feature enables workspace administrators to control the level of access that other administrators have in their workspace.

Previously, admins had default access to all workspaces within an organization. Starting this release, administrators must be explicitly invited and granted roles within each workspace.

This shift enables finer-grained access control, promotes role-based governance, and aligns with the security expectations of enterprise environments.

### Prompt-to-Workflow Generator (Beta)

Simplify workflow creation with AI-driven recommendations, reducing manual effort and accelerating prototyping.

This early beta allows users to describe integration goals in plain language and automatically generate a workflow scaffold.

While activity configuration and data mapping aren't supported yet, this marks a key step toward faster, lower-friction integration design.

We welcome feedback to improve prompt handling, structure quality, and design accuracy.

### Flagged features Now Default

#### Multi-Context Support for Workflows

Workflow context has been upgraded to support multiple contexts including ABS, SFTP, OAuth, and Engage contexts—within the same workflow.

This allows for more flexible, scalable integration designs across teams supporting multitenancy.

#### Application Manifest Logging with Engage

The Application Manifest  now enables users to define integrations, events, and client-configurable variables, with runtime activity logged via Engage for improved observability and audit trails.

## Known Issues&#x20;

* **Template Import Mappings**\
  Importing templates doesn’t bring in existing mappings. A patch is in progress and will be available soon.
* **Schema Registry Search Layout**\
  The search input in the Schema Registry has extra padding around it, which affects how it displays. We’ll correct the spacing in the next update.
* **User-Managed Vault Access**\
  Access to User-Managed Vaults is currently unavailable. We’re investigating and will restore full access shortly.
* **Git-Enabled App Deletion Refresh**\
  After removing a Git-enabled application in a multi-module repository, the UI needs a manual refresh to show the change. We’ll automate this refresh to improve the experience.

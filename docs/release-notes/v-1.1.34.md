## New and Improved Features

1. **Git Enhancements**\
   Improvements to streamline Git workflows and app management.\
   Includes:

* Branch switching now supports stashing or discarding unsaved changes.
* Users can remove applications from both the workspace and local store.
* Broken references during application cloning are now surfaced with clear notifications.
* Git now supports bundle applications for improved versioning and modularity.

2. **Manifest Enhancements**\
   Expanded support in manifest configurations:

* Added support for **levels** and **instance listing**.
* AI-generated documentation is now available directly from the manifest, improving visibility and handoff.

3. **Developer Experience & Productivity**

* The **MultiTab Side Menu** now opens directly in a new tab, improving navigation.\
  **Metrics and Deployment Dashboards** optimized for faster context switching.
* Metrics can now be viewed in **full screen** mode for better readability during monitoring.

4. **Execution Logging (Beta)**\
   Introduced **real-time logs** for workflow execution. Users can now observe log streams live during workflow runs for improved debugging and traceability.
5. **Organization-Based Git Access**\
   Enhanced the Create Application modal to support **organization-specific Git controls**, improving governance and access control during app creation.
6. **Session Expiry UX Update**\
   Users now receive a **5-minute buffer notification** before session expiration, helping avoid mid-edit disruptions.

## Bug Fixes

1. **REST Service Activity Name**\
   Resolved an issue where the name of a REST Service activity was not visible due to duplicate names in the same cluster.
2. **Transition Line Update in Mapper**\
   Transition line conditions now **automatically update** when an activity is renamed within the mapper.
3. **Workflow Duplication Reference**\
   Fixed missing references when duplicating workflows — all dependencies are now correctly carried forward.

---
title: "Importing a Workflow"
sidebar_position: 5
---


You can flexibly import a workflow directly at the Applications level, or in a Workflow folder inside an application, restoring its activities, configurations, and data mappings at the level at which you import it.

To import a workflow:

1. Navigate to the Applications panel.
2. Click the **Add (+)** icon in the top-right corner of the Applications panel and, from the options that appear, select **Import**. You can also right-click on an application and select **Import**.\
   The Import Files modal appears.
3. Click and drag the exported workflow ZIP file that you want to import as an application, or click the central area of the modal to browse and select the workflow ZIP file from your system.\
   The modal refreshes to display the contents of the ZIP file you imported.
4. Specify whether you want to import the workflow into an **Existing Application**, or as a **New Application** by selecting the appropriate option. &#x20;
5. If you want to import the workflow into an existing application, click the **Select Application** field and choose the application into which you want to import the workflow from the options that appear.\
   \
   If you want to import the workflow as a new application, enter the **Application Name** in the field provided.
6. You can view the contents of the ZIP file you imported in the lower half of the Import Files modal. Review this list and deselect the items that you don't want to import.

:::info
Removing these artifacts may break any connections from other assets that rely on them.
:::

7. Click **Import**.\
   The files you selected are imported, and the Review Import modal appears.
8. Review the contents of the workflow you imported and click **Close** to confirm.
